# Dobby 多比 - 智能图片修复小精灵

![Product Showcase](docs/product_showcase.png)

Dobby 多比 是一款功能强大的本地化图片处理工具，结合了最新的计算机视觉和深度学习技术，提供从手动涂抹去水印、AI 智能修复，到透明背景导出和微信表情包制作的一站式工作流。它不仅支持普通图片，还能完美处理带有**透明背景 (Alpha 通道)** 的图片。

## ✨ 核心功能

### 1. 基础模式 (手动涂抹)
适合简单的修补任务或对 AI 结果微调。
- **操作方式**：使用画笔工具手动涂抹需要去除的水印区域。
- **技术原理**：基于 OpenCV 的 Telea 算法（快速行进法），通过计算周围像素的加权平均值来修补缺失区域。

### 2. AI 高级模式 (全自动)
一键识别并去除水印，无需手动操作。
- **多种检测算法**：
    - **文本检测 (EasyOCR)**：专门针对文字水印优化，支持中英文识别，定位精准。
    - **智能检测 (Florence-2)**：使用微软 Florence-2 视觉大模型，能识别 Logo、图标、印章等复杂水印物体。
- **智能修复质量**：
    - **标准**：快速生成遮罩并修复，适合快速预览。
    - **极致 (SAM 2)**：集成 Meta 的 **SAM 2 (Segment Anything Model 2)**，对检测到的区域进行像素级分割，生成极其贴合边缘的遮罩，最大程度保护背景纹理。

### 3. 透明背景完美支持
- **Alpha 通道保护**：独有的分层修复技术。系统会自动分离 RGB 颜色层和 Alpha 透明层，分别进行修复后再无缝合成。
- **结果**：去除水印的同时，**绝对保持图片背景的透明属性**，不会出现黑底或白底问题。

### 4. 微信表情包制作
- **多图批量处理**：支持一次添加多张图片，一键批量输出 `240 × 240` 表情包 PNG。
- **单图规格生成**：可基于选中的单张图片生成以下素材：
    - 横幅：`750 × 400`
    - 封面：`240 × 240`
    - 图标：`50 × 50`
- **自动命名**：导出文件会自动保留原始文件名，并追加规格后缀：
    - `原始名称_240.png`
    - `原始名称_横幅.png`
    - `原始名称_封面.png`
    - `原始名称_图标.png`
- **本地导出**：微信表情包素材在前端浏览器中本地生成并下载，不依赖额外后端接口。

## 🛠️ 技术原理与架构

本项目的核心流程如下：

1.  **图像预处理**：
    - 自动识别图像格式，区分 RGB 和 RGBA (透明) 图像。
    - 在 AI 处理前进行色彩空间转换 (BGR -> RGB)，防止颜色反转。

2.  **水印检测 (Detection)**：
    - **EasyOCR**：通过深度学习文本检测模型定位文字坐标。
    - **Florence-2**：利用 VLM (视觉语言模型) 的物体检测能力，通过 `<OD>` 提示词定位水印/Logo。

3.  **遮罩细化 (Refinement)**：
    - 初步检测得到的通常是矩形框 (Bounding Box)。
    - **SAM 2** 接收这些矩形框作为提示 (Prompt)，输出精细的物体轮廓遮罩，消除矩形框修复时可能带来的背景模糊。

4.  **图像修复 (Inpainting)**：
    - **LaMa (Large Mask Inpainting)**：(如果安装) 使用快速傅里叶卷积网络，擅长处理大面积缺失和恢复周期性纹理。
    - **OpenCV Telea**：(默认兜底) 经典的图像修复算法，稳定可靠。

## 📋 环境要求 (Prerequisites)

- **Python 版本**：推荐使用 **Python 3.12.x**。
    - 虽支持 3.9+，但 3.12 目前对 `torch` 和 `opencv` 等 AI 库的兼容性最稳定。
    - *注意：Python 3.14+ 等极新版本可能导致部分编译库安装失败。*
- **包管理**：建议使用 `venv` 或 `conda` 隔离环境。

## 🚀 快速开始

### 1. 环境准备 (推荐)
在项目根目录下操作：
```bash
# 创建虚拟环境
python3 -m venv .venv

# 激活环境 (macOS/Linux)
source .venv/bin/activate
```
激活后，命令行提示符前应显示 `(.venv)`。

### 2. 前端启动
```bash
cd frontend
npm install
npm run dev
```

如需修改前端请求地址，可在 `frontend` 目录创建 `.env.local`：
```bash
VITE_API_BASE_URL=http://127.0.0.1:8000
```
项目中也提供了示例文件：
```bash
cp .env.example .env.local
```

### 3. 后端启动
```bash
cd backend
# 建议先升级 pip
pip install --upgrade pip

# 安装依赖项
# 若因镜像源导致 torch 版本过低，请参考下方的常见问题
pip install -r requirements.txt

# 启动服务器
python main.py
```

## ❓ 常见问题排查 (Troubleshooting)

### 1. `externally-managed-environment` (macOS)
如果在 `pip install` 时遇到该错误：
- **原因**：macOS 保护系统路径不被修改。
- **解决**：请务必进入并**激活虚拟环境** (`source .venv/bin/activate`) 后再执行安装。

### 2. `torch>=2.5.1` 找不到匹配版本
如果使用了国内镜像源（如清华 TUNA），可能因同步延迟报错：
- **解决**：绕过镜像直接安装官方原版：
  ```bash
  pip install torch torchvision --index-url https://pypi.org/simple
  ```

- **依赖安装失败 (Python 3.14+ / Intel Mac)**
- **现象**：报错中出现 `Failed building wheel` 或 `torch>=2.5.1` 找不到。
- **原因**：PyTorch 官方停止了对 macOS x86_64 的新版本支持。
- **建议**：
    - 使用 Python 3.12 基准。
    - 后端 `requirements.txt` 中已默认注释掉 `sam2`，Intel Mac 用户将无法使用“极致 (Ultra)”修复模式，但标准和 AI 智能检测模式依然可用。

## 📦 依赖模型说明

为了获得最佳效果，系统会尝试加载以下模型：
- **EasyOCR 模型**：首次运行时自动下载。
- **Florence-2-base**：首次使用智能检测时自动下载 (约 1GB)。
- **SAM 2 (Hiera Base+)**：(极致模式专用) 需要手动下载权重文件 `sam2_hiera_base_plus.pt` 放置于 backend 目录。
    - *注意：由于 PyTorch 在 macOS x86_64 (Intel) 上停止更新，Intel Mac 暂时无法安装 SAM 2，极致模式将自动回退到标准模式。*
- **LaMa**：如需高质量修复，需确保 iopaint 及其模型正确安装。
